Cruel Angel's Thesis

This project was done for fun and also to determine the feasibility of using CSS linear gradients to represent display fluid pixel art with more than 10,000 pixels.



■ Visit it here.



According to the CSS Images Module Level 3 specification, the value "crisp-edges" should be used for the image-rendering propery when scaling pixel art. However, at the time of writing, Edge does not support this value. While some browsers roll with their own non-standard name for this value: "-webkit-optimize-contrast" for Chrome and Safari and "-o-crisp-edges" for Opera. Then we have Internet Explorer with its own "-ms-interpolation-mode: nearest-neighbor" declaration for this behavior. Thus I felt that this property value is unsuitable for production sites at the moment.


Currently, the box-shadow property is most often used to represent pixel art but it doesn't scale as well as linear gradient performance-wise. For one, linear gradients can express neighbouring pixels of the same color together as one color-stop while box shadow requires each pixel to have its own value. Besides that, though my own testing on Jsperf, each additional box shadow value also incurs a higher performance hit than each additonal color-stop in a linear gradient. Here, you can see that for this project, a linear gradient solution is almost 300% faster than its box shadow counterpart.


For this project, the source image comes from episode 19 of the Evangelion series. GIMP and Pixelator were used to convert it into pixel art. Next, Javier Borquez's img2css tool is used to convert the pixel art into box-shadow values. Finally, it is fed into my own custom script that converts each column into its own linear gradient and optimises them by merging all vertical neighbours of the same colour into a single color-stop.
